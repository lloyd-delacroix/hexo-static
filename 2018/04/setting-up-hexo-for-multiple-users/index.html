Setting Up Hexo for Multiple Users

Posted on 2018-04-07 | In Jeremiah

This is a short guide on how to set up your hexo installation to easily allow multiple users on the server to write posts and publish. The advantage of this method is it prevents users from accidentally or intentionally editing or deleting posts from other users.

Create a new linux group

The first step is to create a new group that will own all of our hexo files.

After this, you’ll need to close your session and then log back in to refresh the new permissions.

Update permissions on Hexo

Now we need to go through and update all of hexo’s permissions. I’m going to have them all owned by the root user, but you can have them owned by any other user too.

/path/to/hexo should point to the directory where you’re running your Hexo blog from.

As a side note, make sure you have the Hexo folder in a location that all of your users can get to such as /var/www, and not in your home folder. If needed, you can move the entire folder to another location.

Publishing content

To ensure that new files are created as the hexo user, we can use the linux sg command for all of our hexo generates.

Fixing permissions

One problem we still have is after a ‘hexo clean’ the p the db.json files will be deleted, and the next ‘hexo generate’ will recreate it as the current user even when sg is used.

Additionally, sometimes a user may forget to use sg and set the owner to themselves.

To get around this, we’ll create a quick script to fix permissions, called fix_permissions.sh

Authoring posts

For now, I’m using categories to distinguish different post authors. There are some plugins out there to add authors as new front matter, but I have yet to have much luck with any that I’ve tried.

Next steps

There’s a bit more I have planned to try and make this a little smoother. I’ll come back and update this post as I get more things working.
